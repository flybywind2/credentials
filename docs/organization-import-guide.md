# 조직 CSV Import 가이드

조직관리 CSV import는 별도의 `org_type` 컬럼을 받지 않습니다. 시스템은 CSV의 `팀명`과 `그룹명` 값이 비어 있는지에 따라 조직 유형과 승인 경로를 자동 판단합니다.

## CSV 필수 헤더

아래 12개 헤더가 모두 필요합니다. 순서는 템플릿과 동일하게 유지하는 것을 권장합니다.

```csv
실명,실장명,실장ID,팀명,팀장명,팀장ID,그룹명,그룹장명,그룹장ID,파트명,파트장명,파트장ID
```

템플릿 파일은 [organization-import-template.csv](templates/organization-import-template.csv)에 있습니다.

## 직속 조직 판단 기준

| 조직 유형 | CSV 입력 방식 | 승인 경로 |
| --- | --- | --- |
| 일반 조직 `NORMAL` | `팀명` 있음, `그룹명` 있음 | 그룹장 → 팀장 → 실장 |
| 팀 직속 `TEAM_DIRECT` | `팀명` 있음, `그룹명` 비움 | 팀장 → 실장 |
| 실 직속 `DIV_DIRECT` | `팀명` 비움, `그룹명` 비움 | 실장 |

팀 직속 파트는 `그룹명`, `그룹장명`, `그룹장ID`를 모두 비웁니다. 실 직속 파트는 `팀명`, `팀장명`, `팀장ID`, `그룹명`, `그룹장명`, `그룹장ID`를 모두 비웁니다.

## 예시

```csv
실명,실장명,실장ID,팀명,팀장명,팀장ID,그룹명,그룹장명,그룹장ID,파트명,파트장명,파트장ID
데이터전략실,김실장,div001,플랫폼팀,이팀장,team001,AI그룹,박그룹장,group001,AI전략실행파트,최파트장,part001
데이터전략실,김실장,div001,플랫폼팀,이팀장,team001,,,,보안운영파트,정파트장,part002
데이터전략실,김실장,div001,,,,,,,실직속파트,한파트장,part003
```

첫 번째 행은 일반 조직입니다. 승인 경로는 `group001 → team001 → div001`입니다.

두 번째 행은 팀 직속 파트입니다. 그룹 관련 3개 칸이 비어 있으므로 승인 경로는 `team001 → div001`입니다.

세 번째 행은 실 직속 파트입니다. 팀과 그룹 관련 6개 칸이 모두 비어 있으므로 승인 경로는 `div001`입니다.

## 주의 사항

- 빈 값을 입력할 때도 CSV 컬럼은 유지해야 합니다. 값을 지우더라도 콤마(`,`)는 남겨야 컬럼이 밀리지 않습니다.
- `팀명`은 비어 있고 `그룹명`만 있는 행은 잘못된 데이터입니다. 그룹은 팀 아래에 있어야 합니다.
- 팀 직속이면 그룹 관련 3개 칸을 모두 비우고, 실 직속이면 팀/그룹 관련 6개 칸을 모두 비우세요.
- 이메일 컬럼은 입력하지 않습니다. 시스템이 `{사번ID}@samsung.com` 형식으로 자동 표시합니다.
- 사번ID는 승인자 권한 매핑과 이메일 주소 생성에 사용되므로 실제 SSO 사번과 동일하게 입력해야 합니다.
