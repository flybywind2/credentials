# 📋 PRD (Product Requirements Document)

## 1. 개요

### 1.1 문서 정보

| 항목 | 내용 |
|------|------|
| 서비스명 | 부서별 업무 기밀 분류 관리 시스템 (이하 "기밀분류시스템") |
| 버전 | v1.1 |
| 작성일 | 2026-04-21 |
| 작성자 | AI전략실행파트 |
| 대상 조직 | 삼성전자 DS부문 |

### 1.2 배경 및 목적

현재 DS부문 내 부서별 업무에 대한 기밀 여부, 국가핵심기술 해당 여부, Compliance 이슈 연관 여부를 체계적으로 판단·관리하는 시스템이 존재하지 않는다. 본 서비스는 191개 파트의 약 5,000건 세부업무에 대해 기밀 분류 데이터를 빠르게 수집하고, 조직 계층(실 → 팀 → 그룹 → 파트)에 따른 다단계 승인 프로세스를 통해 데이터의 신뢰성을 확보하는 것을 목적으로 한다.

### 1.3 핵심 페인 포인트

기밀 분류 체계가 부재한 상황에서 빠른 시간 내 전 조직의 데이터를 수집해야 하며, 각 부서별 상급자의 Confirm(승인)을 체계적으로 받아야 한다는 점이 가장 큰 과제이다. 수기 관리나 이메일 기반 수집은 누락, 지연, 버전 관리 문제를 야기하므로 웹 기반 시스템을 통해 이를 해결한다.

### 1.4 핵심 목표

본 시스템이 달성해야 할 핵심 목표는 다음과 같다. 첫째, 전 조직(191개 파트)의 세부업무 기밀 분류 데이터를 마감일 내 100% 수집한다. 둘째, 조직 계층별 다단계 승인을 통해 데이터 정확성과 신뢰성을 확보한다. 셋째, 실시간 대시보드를 통해 관리자가 전체 진행 현황을 파악하고 미입력/미승인 조직을 즉시 식별할 수 있도록 한다. 넷째, 사용자 친화적 UX를 통해 빠르고 정확한 데이터 입력을 유도한다.

## 2. 사용자 및 역할 정의

### 2.1 역할 구분

본 시스템은 3가지 역할로 구분되며, 관리자와 승인자가 동일 인물인 경우도 허용한다.

**관리자(Admin):** 시스템 전반을 관리하는 역할이다. 조직 구조(CSV import 및 UI 관리), 기밀 해당 문항/국가핵심기술 판단 문항의 추가·삭제, 각 컬럼 예시 문구 관리, 입력 마감일 설정, 전체 데이터 조회 및 대시보드 확인, Excel Export 기능을 수행한다.

**입력자(파트장):** 본인 파트의 세부업무 데이터를 입력하는 역할이다. 소파트, 대업무, 세부업무를 추가하고, 각 세부업무에 대해 기밀 해당 문항 체크, 국가핵심기술 판단 문항 체크, Compliance 이슈 해당 여부를 입력한다. 파트 단위로 일괄 작성 후 승인 요청하며, 동일 그룹 내 타 파트 데이터를 조회(읽기 전용)할 수 있다.

**승인자(그룹장/팀장/실장):** 산하 조직이 입력한 데이터를 확인하고 승인 또는 반려하는 역할이다. 본인 산하 조직 데이터만 조회 가능하며, 반려 시 사유를 입력한다.

### 2.2 인증

사내 SSO broker 연동을 통해 인증하며, SSO 인증 시 조직 정보와 역할이 자동으로 매핑된다.

## 3. 조직 구조

### 3.1 조직 계층

조직은 실 → 팀 → 그룹 → 파트의 4단계 계층으로 구성된다. 다만, 실 직속(팀/그룹 없이 실에 직접 소속), 팀 직속(그룹 없이 팀에 직접 소속) 형태도 존재한다.

### 3.2 조직 데이터 관리 【변경】

초기 데이터는 CSV import를 통해 일괄 등록하며, 이후 관리자가 UI에서 추가/수정/삭제할 수 있다. CSV에는 각 직책자의 이름과 사번(ID)을 포함하며, 이메일 주소는 `{사번ID}@samsung.com` 규칙으로 자동 생성한다.

**CSV 컬럼 구조:**

| 실명 | 실장명 | 실장ID | 팀명 | 팀장명 | 팀장ID | 그룹명 | 그룹장명 | 그룹장ID | 파트명 | 파트장명 | 파트장ID |
|------|--------|--------|------|--------|--------|--------|----------|----------|--------|----------|----------|

## 4. 핵심 기능 요구사항

### 4.1 데이터 입력

**4.1.1 입력 화면 구성:** 메인 화면은 스프레드시트 형태로 여러 행을 한눈에 볼 수 있으며, 개별 행 클릭 시 상세 입력 폼이 팝업으로 표시된다. 1920x1080 해상도에 최적화하여 핵심 컬럼을 최대한 한 화면에 표시하되, 전체 컬럼이 약 20개이므로 가로 스크롤을 지원한다.

**4.1.2 입력 항목:** 입력 항목은 크게 업무 정보, 기밀 여부, 국가핵심기술, Compliance 이슈, 데이터 보관/저장 정보의 5개 영역으로 구분된다.

업무 정보 영역에는 소파트(텍스트), 대업무(텍스트), 세부업무(텍스트)가 포함된다.

기밀 여부 영역에는 기밀 해당 문항 체크(객관식 복수선택, 필수, 모든 문항에 "해당 없음" 선택지 포함), 기밀 여부 자동 판정, 데이터 유형(자유 텍스트, 기밀 해당 시 필수), 소유자 vs 사용자(단일 선택, 기밀 해당 시 필수)가 포함된다.

**【변경】 기밀 여부 판정 규칙:** 모든 문항에서 "해당 없음"만 선택한 경우 "비기밀"로 판정한다. 1개 문항이라도 "해당 없음" 이외의 선택지를 선택한 경우 "기밀"로 판정한다.

국가핵심기술 영역에는 국가핵심기술 판단 문항 체크(객관식 복수선택, 필수, 모든 문항에 "해당 없음" 선택지 포함), 국가핵심기술 여부 자동 판정, 데이터 유형(자유 텍스트, 해당 시 필수), 소유자 vs 사용자(단일 선택, 해당 시 필수)가 포함된다.

**【변경】 국가핵심기술 판정 규칙:** 기밀 여부 판정과 동일하다. 모든 문항에서 "해당 없음"만 선택하면 "비해당", 그 외에는 "해당"으로 판정한다.

Compliance 이슈 연관 영역에는 해당 여부(직접 체크), 데이터 유형(자유 텍스트, 해당 시 필수), 소유자 vs 사용자(단일 선택, 해당 시 필수)가 포함된다.

데이터 보관/저장 정보 영역에는 데이터 보관 장소/원천서비스(텍스트), 관련 메뉴(텍스트), 공유범위(드롭다운 단일 선택: 부문, 사업부, 실/팀/그룹)가 포함된다.

**4.1.3 컬럼 예시 툴팁:** 각 컬럼 헤더에 마우스 호버 시 해당 컬럼의 입력 예시를 툴팁으로 표시한다. 예시 내용은 관리자가 별도 메뉴에서 입력/수정할 수 있다.

**4.1.4 Excel 연동 기능:** 시스템에서 표준 Excel 양식을 다운로드할 수 있다. Excel에서 작성한 데이터를 클립보드(Ctrl+C)로 복사 후 웹 화면에 붙여넣기(Ctrl+V)하면 미리보기 화면이 표시된다. 미리보기 화면에서 데이터 검증을 수행하며, 필수 입력 누락, 형식 오류 등이 있는 행을 오류 표시(빨간색 하이라이트 + 오류 메시지)한다. 오류 없는 행만 선택 저장하거나, 오류 수정 후 전체 저장할 수 있다. 별도의 Excel 파일 Import(파일 업로드) 기능도 제공한다.

### 4.2 승인 프로세스

**4.2.1 승인 경로:** 조직 유형에 따라 승인 단계가 달라진다. 일반 파트는 파트장(입력) → 그룹장(확인) → 팀장(확인) → 실장(확인) 순서이다. 팀 직속은 담당자(입력) → 팀장(확인) → 실장(확인) 순서이다. 실 직속은 담당자(입력) → 실장(확인) 순서이다.

**4.2.2 승인 단위:** 파트 단위로 전체 세부업무를 일괄 작성 후 한 번에 승인 요청한다.

**4.2.3 승인/반려/취소:** 승인자는 산하 조직의 데이터를 검토 후 승인 또는 반려할 수 있다. 반려 시 사유 입력은 필수이며, 반려된 건은 어느 단계에서 반려되든 최초 입력자(파트장)에게 돌아간다. 승인 처리가 시작되기 전에는 요청자 또는 관리자가 승인 요청을 취소할 수 있고, 취소된 건은 다시 승인 요청할 수 있다. 파트장은 수정 후 다시 승인 요청할 수 있다.

**4.2.4 승인 완료 후 수정:** 최종 승인 완료된 데이터도 파트장이 수정 요청 가능하며, 수정 시 승인 프로세스를 다시 처음부터 진행한다.

### 4.3 이메일 알림 【변경】

사내 mail API를 활용하여 아래 시점에 이메일 알림을 발송한다. 이메일 주소는 `{사번ID}@samsung.com` 형식이다.

승인 요청 시 다음 단계 승인자에게 알림을 보낸다. 승인 완료 시 다음 단계 승인자에게 알림을 보내며, 각 단계 승인 시 관리자에게도 알림을 보낸다. 최종 승인 완료 시 파트장에게 완료 알림을 보낸다. 반려 시 파트장에게 반려 알림(반려 사유 포함)을 보낸다.

### 4.4 관리자 기능

**4.4.1 대시보드:** 관리자 대시보드에는 부서별 입력 완료율(입력 완료 파트 수 / 전체 파트 수), 승인 진행 현황(대기/진행중/승인완료/반려 건수), 기밀 업무 비율(전체 세부업무 대비), 국가핵심기술 해당 업무 수, Compliance 이슈 해당 업무 수, 부서별/파트별 진행 상태 요약 테이블, 전체 업무 대비 기밀/국가핵심기술/Compliance 이슈 통합 비율, 입력 마감일 D-day 표시가 포함된다.

**4.4.2 전체 데이터 조회:** 스프레드시트 형태로 전체 데이터를 조회할 수 있으며, 부서별, 파트별, 승인 상태별 필터링 기능을 제공한다.

**4.4.3 문항 관리 【변경】:** 기밀 해당 문항과 국가핵심기술 판단 문항을 추가/삭제할 수 있다. 각 문항은 객관식 복수선택 형태이며, 모든 문항에 "해당 없음" 선택지가 기본 포함된다. 관리자가 문항 추가 시 "해당 없음"은 시스템이 자동으로 포함시킨다.

**4.4.4 예시 관리:** 각 입력 컬럼의 툴팁 예시 문구를 입력/수정할 수 있다.

**4.4.5 마감일 관리:** 입력 마감일을 설정하고, 대시보드에서 D-day를 표시한다.

**4.4.6 Excel Export:** 전체 데이터 일괄 다운로드, 부서별/파트별 필터링 후 다운로드, 승인 상태별 필터링 후 다운로드를 지원한다.

### 4.5 데이터 접근 권한

파트장은 본인 파트 데이터를 읽기/쓰기할 수 있으며, 동일 그룹 내 타 파트 데이터는 읽기 전용으로 조회 가능하다. 승인자(그룹장/팀장/실장)는 본인 산하 조직 데이터만 조회 가능하다. 관리자는 전체 데이터를 조회할 수 있다. 모든 조회 화면에서 필터링 기능을 제공한다.

## 5. UX/UI 요구사항 【변경】

### 5.1 디자인 원칙

무채색 기반의 깔끔한 UI를 적용한다. 색상은 흰색, 회색 계열을 기본으로 하며, 상태 표시(승인/반려/기밀 등)에만 최소한의 포인트 컬러를 사용한다. 사용자 친화적 UX를 우선하며, 데이터 입력의 편의성과 속도에 최적화한다.

### 5.2 해상도

사내 표준 모니터 1920x1080 해상도에 최적화한다. 모바일 및 반응형 대응은 불필요하다. 최소 지원 해상도는 1920x1080이며, 이 해상도에서 가로 스크롤을 최소화하도록 레이아웃을 설계한다.

## 6. 비기능 요구사항

동시 접속 100명을 안정적으로 지원해야 한다. 191개 파트, 약 5,000건 세부업무 데이터를 처리해야 한다. 사내 프라이빗 클라우드에 배포하며, 사내 SSO broker 연동을 통한 인증 보안을 확보한다.
