## 2. 화면별 상세 SPEC

### 2.1 S03 — 데이터 입력 (스프레드시트) 화면 【변경】

**해상도 기준:** 1920x1080에서 좌측 네비게이션(200px) + 메인 콘텐츠(1720px)로 구성한다. 스프레드시트 영역은 1680px 너비(좌우 패딩 20px)를 확보하며, 핵심 컬럼(No, 소파트, 대업무, 세부업무, 기밀여부, 국가핵심기술여부, Compliance여부)을 1680px 내에 표시하고, 나머지 상세 컬럼은 가로 스크롤로 접근한다.

**레이아웃:** 상단에 파트 정보(실 > 팀 > 그룹 > 파트)와 입력 상태(DRAFT/SUBMITTED/APPROVED/REJECTED), 마감일 D-day를 표시한다. 반려된 경우 반려 사유를 상단 알림 배너로 표시한다. 중단에 스프레드시트 형태의 데이터 테이블을 배치하며, 고정 컬럼(No, 소파트, 대업무, 세부업무)과 스크롤 가능 컬럼(기밀 관련, 국가핵심기술 관련, Compliance 관련, 데이터 보관 관련)으로 구분한다. 각 컬럼 헤더에 (i) 아이콘을 배치하고, 마우스 호버 시 관리자가 입력한 예시 문구를 툴팁으로 표시한다. 하단에 행 추가 버튼, Excel 양식 다운로드 버튼, 클립보드 붙여넣기 버튼, Excel Import 버튼, 저장 버튼, 승인 요청 버튼을 배치한다.

**행 클릭 동작:** 행을 클릭하면 S04 상세 입력 폼이 모달 팝업으로 열린다.

**자동 판정 표시 【변경】:** 기밀 해당 문항에서 "해당 없음" 외 항목이 1개 이상 선택된 문항이 있으면 "기밀 여부" 셀에 빨간 배경의 "기밀" 뱃지를 표시한다. 모든 문항이 "해당 없음"만 선택한 경우 회색 배경의 "비기밀" 뱃지를 표시한다. 국가핵심기술도 동일한 로직으로 "해당"/"비해당" 뱃지를 표시한다.

**필수 입력 검증:** 기밀 해당 문항과 국가핵심기술 판단 문항은 모든 문항에 대해 최소 1개 선택지를 선택해야 한다("해당 없음" 포함). 기밀 판정 시 데이터 유형, 소유자/사용자가 필수가 된다. 국가핵심기술 해당 시 데이터 유형, 소유자/사용자가 필수가 된다. Compliance 이슈 해당 체크 시 데이터 유형, 소유자/사용자가 필수가 된다. 누락 시 해당 셀에 빨간 테두리와 오류 메시지를 표시한다.

**승인 요청:** 승인 요청 버튼 클릭 시 전체 데이터 검증을 수행하고, 오류가 있으면 오류 행을 하이라이트 표시하며 제출을 차단한다. 검증 통과 시 S06 확인 화면으로 이동한다.

### 2.2 S04 — 상세 입력 폼 (팝업) 【변경】

**레이아웃:** 모달 팝업(폭 900px, 높이 auto, 최대 높이 900px, 세로 스크롤)으로 열리며, 5개 영역을 섹션으로 구분하여 세로 배치한다.

**업무 정보 섹션:** 소파트(텍스트 입력), 대업무(텍스트 입력), 세부업무(텍스트 입력) 필드를 배치한다.

**기밀 여부 섹션 【변경】:** 관리자가 등록한 기밀 해당 문항 목록을 표시하고, 각 문항에 대해 객관식 선택지를 체크박스(복수선택)로 제공한다. 모든 문항에 "해당 없음" 선택지가 포함되어 있다. "해당 없음"을 선택하면 해당 문항의 다른 선택지는 자동 해제되고, 다른 선택지를 선택하면 "해당 없음"이 자동 해제된다(상호 배타적). 모든 문항 응답은 필수이다. 모든 문항에서 "해당 없음"만 선택된 경우 상단에 회색 "비기밀" 뱃지가 표시되고, 데이터 유형/소유자·사용자 필드가 비활성화된다. 1개라도 "해당 없음" 외 선택지가 있으면 상단에 빨간 "기밀" 뱃지가 표시되고, 데이터 유형(텍스트 입력, 필수)과 소유자/사용자(라디오 버튼, 필수) 필드가 활성화된다.

**국가핵심기술 섹션 【변경】:** 기밀 여부 섹션과 동일한 구조 및 "해당 없음" 상호 배타 로직을 적용한다.

**Compliance 이슈 섹션:** 해당 여부 토글/체크박스, 해당 시 데이터 유형과 소유자/사용자 필드가 활성화된다.

**데이터 보관 정보 섹션:** 데이터 보관 장소(텍스트), 관련 메뉴(텍스트), 공유범위(드롭다운: 부문/사업부/실·팀·그룹) 필드를 배치한다.

**하단 버튼:** 저장(현재 행 저장 후 팝업 유지), 저장 후 닫기(저장 후 팝업 닫기), 취소(변경 사항 미저장 후 닫기)를 배치한다.

### 2.3 S05 — 데이터 붙여넣기 미리보기

**진입 경로:** S03에서 클립보드 붙여넣기 버튼 클릭 또는 Ctrl+V 감지, 또는 Excel Import(파일 업로드) 시 이 화면으로 전환된다.

**레이아웃:** 상단에 안내 문구("아래 데이터를 확인하세요. 오류가 있는 행은 빨간색으로 표시됩니다.")를 배치한다. 중단에 미리보기 테이블을 표시하며, 정상 행은 기본 스타일, 오류 행은 빨간 배경 + 해당 셀에 오류 아이콘과 메시지를 표시한다. 하단에 오류 요약(총 N건 중 정상 M건, 오류 K건), 전체 저장(오류 행 제외하고 정상 행만 저장) 버튼, 취소 버튼을 배치한다.

**검증 항목:** 필수 컬럼 누락 여부, 드롭다운 값의 유효성(공유범위, 소유자/사용자), 데이터 형식 오류를 검증한다.

### 2.4 S07 — 승인 대기 목록

**레이아웃:** 내 승인 대기 건을 카드 또는 테이블 형태로 표시한다. 각 항목에 요청 조직(파트명), 요청자, 요청일시, 세부업무 건수, 현재 상태를 표시한다. 클릭 시 S08 승인 상세 검토 화면으로 이동한다.

### 2.5 S08 — 승인 상세 검토

**레이아웃:** 상단에 조직 정보와 승인 이력(각 단계별 상태 타임라인)을 표시한다. 중단에 해당 파트의 전체 업무 데이터를 스프레드시트 형태(읽기 전용)로 표시한다. 행 클릭 시 상세 내용을 읽기 전용 팝업으로 확인할 수 있다. 하단에 승인 버튼과 반려 버튼을 배치하며, 반려 클릭 시 반려 사유 입력 모달이 표시되고 사유 입력은 필수이다.

### 2.6 S09 — 관리자 대시보드

**레이아웃:** 1920x1080 해상도 전체를 활용한다. 상단에 마감일 D-day 배너, 전체 입력 완료율 프로그레스 바를 배치한다. 카드 영역에 주요 지표 카드를 배치한다: 기밀 업무 비율, 국가핵심기술 해당 업무 수, Compliance 이슈 해당 업무 수, 기밀/국가핵심기술/Compliance 통합 비율. 차트 영역에 승인 진행 현황 도넛 차트(대기/진행중/승인완료/반려)를 배치한다. 하단에 부서별/파트별 진행 상태 요약 테이블을 배치하며, 필터링과 정렬을 지원한다.

### 2.7 S10 — 전체 데이터 조회

**레이아웃:** 상단에 필터 바를 배치한다. 필터 항목은 실, 팀, 그룹, 파트(계층적 드롭다운), 승인 상태, 기밀 여부, 국가핵심기술 여부, Compliance 이슈 여부이다. 중단에 전체 데이터 스프레드시트(읽기 전용)를 표시한다. 하단에 Excel 다운로드 버튼(현재 필터 조건 반영)을 배치한다.

### 2.8 S11 — 조직 관리 【변경】

**레이아웃:** CSV Import 영역(파일 업로드 + 미리보기 + 등록)과 조직 목록 테이블(검색, 추가, 수정, 삭제)을 배치한다. CSV Import 시 미리보기에서 실명, 실장명, 실장ID, 팀명, 팀장명, 팀장ID, 그룹명, 그룹장명, 그룹장ID, 파트명, 파트장명, 파트장ID를 확인할 수 있으며, 이메일 컬럼에는 `{ID}@samsung.com`이 자동 표시된다. 조직 추가/수정 폼에서도 각 직책자의 이름과 사번ID를 입력하며, 이메일은 자동 생성 규칙으로 표시한다(편집 불가).

### 2.9 S12 — 문항 관리 【변경】

**레이아웃:** 탭으로 기밀 해당 문항과 국가핵심기술 판단 문항을 구분한다. 각 탭에서 문항 목록을 표시하고, 문항 추가(문항 텍스트 + 객관식 선택지 N개 입력) 및 삭제 기능을 제공한다. 문항 추가 시 "해당 없음" 선택지는 시스템이 자동으로 포함시키며, 관리자에게 안내 문구("'해당 없음' 선택지는 자동으로 추가됩니다")를 표시한다. 드래그 앤 드롭으로 문항 순서를 변경할 수 있다.

### 2.10 S13 — 예시 관리

**레이아웃:** 전체 입력 컬럼 목록을 테이블로 표시하고, 각 컬럼의 예시 문구를 인라인 편집할 수 있다. 저장 시 즉시 반영되어 사용자 화면의 툴팁에 표시된다.

### 2.11 S14 — 마감일 관리

**레이아웃:** 날짜 선택기(Date Picker)와 설명 입력 필드, 저장 버튼을 배치한다. 현재 설정된 마감일과 D-day를 표시한다.

## 3. 비즈니스 로직 상세

### 3.1 기밀 여부 자동 판정 【변경】

기밀 해당 문항이 N개 있을 때, 각 문항에는 관리자가 설정한 선택지와 시스템이 자동 추가한 "해당 없음" 선택지가 포함된다. 모든 문항에서 반드시 1개 이상의 선택지를 선택해야 한다(필수 응답). "해당 없음"과 다른 선택지는 상호 배타적으로, "해당 없음" 선택 시 해당 문항의 다른 선택지는 해제되고, 다른 선택지 선택 시 "해당 없음"은 해제된다.

판정 규칙은 다음과 같다. 전체 N개 문항 모두에서 "해당 없음"만 선택한 경우 해당 세부업무는 **"비기밀"**로 판정한다. 1개 문항이라도 "해당 없음" 이외의 선택지가 선택된 경우 **"기밀"**로 판정한다. 기밀 판정 시 데이터 유형(자유 텍스트)과 소유자/사용자(단일 선택) 입력이 필수가 된다.

### 3.2 국가핵심기술 여부 자동 판정 【변경】

기밀 여부 판정과 완전히 동일한 로직을 적용한다. 모든 문항에서 "해당 없음"만 선택하면 "비해당", 1개라도 "해당 없음" 외 선택지가 있으면 "해당"으로 판정한다.

### 3.3 승인 경로 결정

조직의 org_type에 따라 승인 경로를 자동 결정한다. NORMAL 유형은 그룹장 → 팀장 → 실장 3단계, TEAM_DIRECT 유형은 팀장 → 실장 2단계, DIV_DIRECT 유형은 실장 1단계이다. 승인 요청 제출 시 organizations 테이블에서 각 직책자의 사번ID를 참조하여 approval_requests와 approval_steps를 자동 생성한다.

### 3.4 승인/반려 처리

승인 시 현재 단계의 status를 APPROVED로 변경하고, approval_requests의 current_step을 증가시키며, 다음 단계 승인자에게 이메일 알림(`{사번ID}@samsung.com`)을 발송하고, 관리자에게도 알림을 발송한다. 최종 단계 승인 시 approval_requests의 status를 APPROVED로 변경하고, 해당 파트의 모든 task_entries의 status를 APPROVED로 변경하며, 입력자와 관리자에게 완료 알림을 발송한다.

반려 시 현재 단계의 status를 REJECTED로 변경하고 reject_reason을 저장하며, approval_requests의 status를 REJECTED로 변경하고, 해당 파트의 모든 task_entries의 status를 REJECTED로 변경하며, 입력자에게 반려 알림(사유 포함)을 발송한다. 입력자가 수정 후 재제출 시 새로운 approval_requests를 생성하여 처음부터 승인 프로세스를 다시 시작한다.

### 3.5 데이터 접근 권한 로직

입력자(파트장)는 본인 organization_id의 데이터에 대해 읽기/쓰기 권한을 가지며, 동일 group_name을 가진 조직의 데이터에 대해 읽기 권한을 가진다. 승인자는 본인 산하 조직(조직 계층에서 하위에 해당하는 모든 조직)의 데이터에 대해 읽기 권한을 가진다. 관리자는 전체 데이터에 대해 읽기/쓰기 권한을 가진다. 모든 API에서 서버 사이드 권한 체크를 수행하며, 프론트엔드에서도 UI 레벨 접근 제어를 병행한다.

### 3.6 클립보드 붙여넣기 처리 로직

사용자가 Ctrl+V 또는 붙여넣기 버튼을 사용하면 클립보드의 탭 구분 텍스트(TSV)를 파싱한다. 파싱된 데이터를 Excel 양식의 컬럼 순서에 매핑하고, 각 행에 대해 필수 입력 검증, 선택지 유효성 검증, 데이터 형식 검증을 수행한다. 검증 결과를 미리보기 화면(S05)에 표시하며, 사용자 확인 후 정상 행만 일괄 저장한다.
