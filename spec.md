# SPEC (Functional Specification Document)

최신 반영일: 2026-04-25

## 1. 범위

기밀분류시스템은 부서별 세부업무의 기밀, 국가핵심기술, Compliance 해당 여부를 입력, 검토, 승인하는 단일 포트 웹 애플리케이션이다. FastAPI가 `/api/*`와 프론트엔드를 함께 제공하며, 모든 주요 페이지는 메일 링크와 직접 접근을 위해 고유 URL을 가진다.

## 2. 역할 및 접근 권한

| 역할 | 주요 권한 | 접근 화면 |
| --- | --- | --- |
| INPUTTER | 본인 파트 업무 입력/수정, 본인이 추가한 행 삭제, 본인 파트 현황 조회, 동일 그룹 읽기 전용 조회 | `/inputter`, `/status`, `/group` |
| APPROVER | 관리자 메뉴를 제외한 입력/현황/그룹/승인 화면 접근, 산하 또는 담당 파트 검토, 개별 항목 승인/반려 의견 작성 | `/inputter`, `/status`, `/group`, `/approver`, `/approver/approvals/{id}` |
| ADMIN | 전체 데이터 조회/관리, 조직/문항/툴팁/마감일/사용자 권한/파트원 관리, 전체 승인 대기 조회 | 전체 화면 |

관리자와 승인자가 동일 인물인 경우를 허용한다. 그룹장이 파트장을 겸임하는 경우 APPROVER로 매핑하되, 본인이 파트장인 파트는 입력/수정 대상 파트로 선택할 수 있다.

## 3. 화면 목록

| ID | URL | 화면명 | 접근 역할 | 설명 |
| --- | --- | --- | --- | --- |
| S01 | `/` | 로그인 | 전체 | SSO/mock 인증 진입점 |
| S03 | `/inputter` | 데이터 입력 | INPUTTER, APPROVER, ADMIN | 파트 업무 입력, Excel/붙여넣기, 승인 요청 |
| S07 | `/approver` | 승인 대기 목록 | APPROVER, ADMIN | 본인 승인 차례 목록, ADMIN은 전체 목록 |
| S08 | `/approver/approvals/{id}` | 승인 상세 검토 | APPROVER, ADMIN | 항목별 검토 후 검토 완료 |
| S09 | `/admin` | 관리자 | ADMIN | 대시보드, 전체 조회, 조직/문항/사용자/파트원 관리 |
| S15 | `/status` | 내 파트 현황 | INPUTTER, APPROVER, ADMIN | 파트 입력/승인 상태와 파트원 명단 |
| S16 | `/group` | 동일 그룹 조회 | INPUTTER, APPROVER, ADMIN | 같은 그룹 파트 업무 읽기 전용 조회 |

## 4. 인증 및 사용자 매핑

개발 모드(`SSO_MODE=mock`)는 로그인 화면의 사번 ID와 `X-Employee-Id` 헤더를 사용한다. stale bearer token이 남아 있어도 mock 모드에서는 명시된 `X-Employee-Id`가 현재 사용자 기준이다.

운영 broker 모드(`SSO_MODE=broker`)는 사내 SSO broker/reverse proxy가 주입한 내부 헤더를 신뢰한다. 기본 사번 헤더는 `X-Broker-Employee-Id`이며, 표시명과 이메일은 `SSO_BROKER_NAME_HEADER`, `SSO_BROKER_EMAIL_HEADER`로 선택 설정한다. broker 모드에서는 개발용 `X-Employee-Id`와 브라우저 bearer token이 권한을 덮어쓰지 않는다.

LDAP/SAML 모드는 인증 성공 후 HMAC 서명 bearer token을 발급하고 API 요청의 `Authorization: Bearer ...`를 신뢰한다. 사용자 역할은 `users` 테이블, `SSO_ADMIN_EMPLOYEE_IDS`, 조직의 파트장/그룹장/팀장/실장 ID 매칭 순으로 결정한다.

## 5. 데이터 입력 화면

데이터 입력 화면은 1920x1080 데스크톱 기준으로 전체 폭과 높이를 채운다. 모바일 전용 UI는 제공하지 않는다. 좌측 네비게이션과 메인 스프레드시트 영역으로 구성하며, 고정 컬럼(No, 소파트, 대업무, 세부업무)과 가로 스크롤 컬럼을 분리한다.

입력자는 본인 파트 데이터를 기본으로 조회한다. 승인자가 하위 파트를 가진 경우 파트 선택 후 해당 파트의 업무를 수정할 수 있다. ADMIN은 전체 조직 대상 데이터를 관리할 수 있다.

행 추가, 행 수정, 저장, 저장 후 닫기, 취소, 삭제, 승인 요청을 제공한다. 삭제는 ADMIN 또는 해당 행을 생성한 사용자만 가능하다. 반려된 행은 항목별 검토 의견과 반려 표시를 우선 노출한다.

## 6. Excel/붙여넣기 입력

Excel 양식 다운로드와 Excel Import는 `소파트`, `대업무`, `세부업무` 3개 컬럼만 받는다. Excel 붙여넣기는 raw TSV 원문을 노출하지 않고 3개 컬럼 그리드형 모달에서 검증 결과를 보여준다. 브라우저는 Excel `text/html` 테이블을 우선 해석하고, HTML이 없으면 `text/plain` TSV를 fallback으로 처리한다. 분류 작업은 반드시 웹 상세 입력 모달에서 수행한다.

업로드된 행은 `UPLOADED` 상태로 저장한다. `UPLOADED` 행이 하나라도 남아 있으면 승인 요청을 차단하고, 사용자가 웹에서 분류 저장을 완료해야 `DRAFT`로 전환된다. 미리보기 화면은 정상/오류 행 수, 선택 저장, 취소를 제공한다.

## 7. 상세 입력 모달

상세 입력 모달은 업무 정보, 담당자 배정, 기밀 여부, 국가핵심기술, Compliance, 데이터 보관 정보 섹션으로 구성한다. 업무 정보 영역의 높이는 고정 규격으로 유지하고, 라디오/체크 컨트롤은 텍스트와 정렬을 맞춘다.

기밀과 국가핵심기술 문항은 각 문항마다 `"해당 없음"` 선택지를 포함한다. 신규 또는 응답이 없는 문항은 `"해당 없음"`을 기본 선택으로 표시한다. `"해당 없음"`과 다른 선택지는 상호 배타적으로 동작한다. 각 섹션에는 `"해당 없음 일괄 선택"` 버튼을 제공한다.

데이터 유형과 소유자/사용자 입력은 항상 가능하다. 다만 기밀, 국가핵심기술, Compliance가 해당 상태인데 필수 메타데이터가 비어 있으면 저장 시 팝업/오류 메시지로 입력을 요구한다.

파트원 배정은 관리자에서 업로드한 파트원 명단을 기준으로 같은 파트의 인원을 드롭다운/체크박스로 표시한다. 사용자는 업무별 담당자를 복수 배정할 수 있다.

## 8. 판정 로직

기밀 문항 중 하나라도 `"해당 없음"` 외 선택지가 있으면 기밀로 판정한다. 모든 문항이 `"해당 없음"`이면 비기밀이다.

국가핵심기술도 동일하게 판정한다. Compliance는 사용자가 해당 여부를 직접 체크한다. 각 항목이 해당 상태이면 데이터 유형과 소유자/사용자 값이 필수이다.

승인 요청 전 전체 업무를 검증한다. 필수 값 누락, `UPLOADED` 상태, 기밀/국가핵심기술/Compliance 메타데이터 누락이 있으면 제출을 차단하고 오류 행을 표시한다.

## 9. 승인 프로세스

승인 경로는 조직 정보에서 실제 직책자 ID를 읽어 생성한다. `NORMAL`은 그룹장 -> 팀장 -> 실장, `TEAM_DIRECT`는 팀장 -> 실장, `DIV_DIRECT`는 실장 순서이다. 실장이 없는 조직은 실장 단계를 생략한다.

승인자는 현재 본인 차례인 요청만 승인 대기 목록에서 본다. ADMIN은 전체 승인 대기 목록을 본다. 승인 상세 URL은 `/approver/approvals/{id}`이며 메일 링크에서 직접 진입할 수 있다.

승인 상세에서는 각 업무 항목에 대해 승인/반려를 개별 선택하고 의견을 입력한다. 반려 항목에는 의견이 필수이다. 화면 상단 버튼은 `검토 완료` 1개만 제공하며, 항목 중 반려가 있으면 반려 처리, 모두 승인되면 승인 처리로 제출한다.

최종 승인 시 승인 요청과 업무 상태를 `APPROVED`로 변경한다. 반려 시 승인 요청과 업무 상태를 `REJECTED`로 변경하고, 입력자는 수정 후 새 승인 요청을 생성한다. 승인 완료 후에도 수정 요청을 통해 재승인 플로우를 시작할 수 있다.

## 10. 내 파트 현황 및 동일 그룹 조회

내 파트 현황은 본인 파트의 입력/승인 진행 상태와 파트원 명단을 표시한다. 파트원 명단 CSV 업로드는 관리자 화면에서만 수행한다. CSV 컬럼은 `파트명`, `이름`, `knox_id`이다.

동일 그룹 조회는 같은 그룹의 파트 업무를 읽기 전용으로 표시한다. 대량 데이터는 페이지네이션을 제공하고, 행 번호는 전체 목록 기준 번호를 유지한다.

## 11. 관리자 기능

관리자 화면은 접기/펴기 가능한 섹션과 페이지네이션으로 스크롤을 최적화한다.

조직 관리는 CSV Import, 미리보기, 추가, 수정, 삭제를 제공한다. 조직 CSV는 실/팀/그룹/파트명과 각 직책자 이름/ID를 포함한다. 실장 정보는 없는 조직도 허용한다. 그룹장과 파트장이 같은 ID인 겸임 조직도 허용한다.

문항 관리는 기밀/국가핵심기술 탭을 제공하고 문항 추가, 삭제, drag-and-drop 순서 변경을 지원한다. 문항 추가 시 `"해당 없음"`은 시스템이 자동 포함한다.

사용자 권한 관리는 사번, 이름, 역할, 조직을 조회/추가/수정/삭제한다. ADMIN만 접근할 수 있다.

파트원 관리는 `파트명`, `이름`, `knox_id` CSV 업로드와 조직별 명단 조회를 제공한다. 이 데이터는 입력 화면의 담당자 배정과 내 파트 현황에 사용된다.

대시보드는 마감일 D-day, 입력 완료율, 기밀/국가핵심기술/Compliance 지표, 통합 비율, 승인 현황 도넛 차트, 부서별 요약 필터/정렬을 제공한다. 전체 조회는 실/팀/그룹/파트 계층 필터와 상태/판정 필터, Excel Export를 제공한다.

## 12. 이메일 알림

승인 요청, 단계 승인, 최종 승인, 반려 이벤트에서 메일 알림을 발송한다. 수신자 주소는 `{사번ID}@samsung.com`으로 생성한다.

사내 메일 API 모드는 `MAIL_API_BASE_URL`에 대해 `POST /send_mail`을 호출한다. 기본 endpoint 예시는 `https://mail.net/send_mail`이다. payload는 JSON 형식의 `recipients: string[]`, `title: string`, `content: html string`만 전달한다. `docSecuType`, `contentType`, `recipientType`, 실제 사내 메일 API header 조립은 `/send_mail` 라우터가 담당한다. 승인 요청 메일에는 `/approver/approvals/{id}` 상세 URL을 포함한다.

## 13. 권한 제어 원칙

모든 권한은 서버 API에서 강제하고, 프론트엔드는 메뉴/버튼 노출을 보조적으로 제한한다.

입력자는 본인 조직 업무만 쓰기 가능하고 동일 그룹 업무는 읽기 전용이다. 승인자는 산하 조직 또는 본인이 승인 단계에 포함된 요청을 조회/검토할 수 있다. ADMIN은 전체 조회와 관리 기능에 접근할 수 있다. 관리자 전용 API는 `/api/admin/*` 경로로 분리한다.

## 14. 운영 및 배포

로컬 개발은 Python 3.10.19 기준이다. 기본 DB는 SQLite이며 운영 DB는 MySQL `DATABASE_URL`로 연결한다. Docker/Kubernetes 배포 파일을 제공하고, 운영 환경에서는 broker/LDAP/SAML 중 사내 표준 인증 방식을 선택한다.

운영에서는 mock 모드와 외부에서 임의 주입 가능한 `X-Employee-Id`를 노출하지 않는다. broker 사용 시 ingress/reverse proxy는 외부 요청의 `X-Broker-*` 헤더를 제거한 뒤 인증된 사용자 헤더만 내부로 재주입해야 한다.
